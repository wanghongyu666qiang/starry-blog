# Starry Blog 使用手册

> 项目路径：`D:\workbuddyh\starry-blog` | 技术栈：Next.js 16 + React 19 + Tailwind CSS 4 | 内容存储：Markdown + JSON 文件驱动

---

## 一、本地开发

### 首次运行（必须先装依赖）

```bash
cd D:\workbuddyh\starry-blog
npm install        # 首次运行必须执行，之后可跳过
```

### 日常开发

```bash
npm run dev        # 启动开发服务器 → http://localhost:3000
npm run build      # 构建生产版本
```

**注意**：项目使用 `--webpack` 而非 Turbopack（解决 Windows 路径问题），已在 `package.json` 中配置好了。

## 二、管理员后台

### 2.1 初始设置

在项目根目录创建 `.env.local`：

```
ADMIN_PASSWORD=你的密码
```

### 2.2 登录

访问 `你的域名/admin/login`，输入密码。

认证方式：HTTP-only Cookie（`admin_token`），7天有效。部署到 Vercel 后自动使用 HTTPS。

### 2.3 后台页面一览

| 页面 | 路径 | 功能 |
|------|------|------|
| 概览 | `/admin/dashboard` | 文章/项目/时间线数量统计 |
| 文章管理 | `/admin/posts` | 列表 + 新建 + 编辑 + 删除 |
| 项目管理 | `/admin/projects` | 列表 + 新建 + 编辑 + 删除 |
| 时间线 | `/admin/timeline` | 增删改所有事件 |
| 设置 | `/admin/settings` | 配置说明（只读） |

### 2.4 写一篇新文章

1. 进入 `/admin/posts/new`
2. 填写表单：

| 字段 | 说明 | 必填 |
|------|------|------|
| 文章名 | 标题，自动生成 URL slug | ✅ |
| 分类 | `Engineering` / `Research` / `Learning` / `Thoughts` | 否 |
| 标签 | 逗号分隔，如 `C++, WebAssembly, 图论` | 否 |
| 日期 | 发布日期 | 否 |
| 简述 | 用于 SEO 和卡片摘要 | 否 |
| 已发布 | 勾选才显示在前台 | 否 |
| Markdown 内容 | 正文，右侧实时预览 | 否 |

3. 点击「保存」

编辑器支持：**代码高亮**、**KaTeX 数学公式**、**Mermaid 图表**、**GFM 表格和任务列表**。

### 2.5 添加一个项目

进入 `/admin/projects/new`，额外字段：

| 字段 | 说明 |
|------|------|
| Role | 你的角色和负责内容，如 `Full-stack Developer — 负责…` |
| 技术栈 | 逗号分隔 |
| GitHub 链接 | 仓库 URL |
| Demo 链接 | 在线演示 URL |
| 状态 | 草稿 / 进行中 / 已完成 |

## 三、文件驱动系统

所有内容以文件形式存储在 `src/content/` 下，可以直接用 Obsidian / VS Code 编辑。

### 3.1 目录结构

```
src/content/
  posts/
    campus-navigation-wasm.md    # 文章
    mas-aggregation-survey.md
    ...
  projects/
    zhi-xue.md                   # 项目
    hainanu-navigation.md
    smart-expense-tracker.md
  timeline.json                  # 时间线事件数组
```

### 3.2 文章 Frontmatter

```yaml
---
title: "用 WebAssembly 构建校园导航系统"
description: "将 C++ 图算法移植到浏览器：架构设计…"
date: "2026-06-15"
category: "Engineering"
tags: ["C++", "WebAssembly", "图论", "架构设计"]
published: true
---
```

**分类体系**：

| 分类 | 含义 | 匹配文章类型 |
|------|------|-------------|
| Engineering | 技术实现 · 架构设计 · 踩坑记录 | 工程项目 |
| Research | 论文阅读 · AI 研究 | 学术/研究 |
| Learning | 数据结构 · 算法 · 课程笔记 | 学习笔记 |
| Thoughts | 开发感悟 · 个人思考 | 随笔 |

- `slug` 由文件名自动生成（去掉 `.md`）
- `published: false` 可在前台隐藏文章

### 3.3 项目 Frontmatter

```yaml
---
title: "智学 (ZhiXue)"
description: "Obsidian AI 知识助手…"
date: "2026-07-01"
role: "Full-stack Developer — 负责…"
tech_stack: ["TypeScript", "Python", "FastAPI", "LightRAG"]
github_url: "https://github.com/wanghongyu666qiang/---obsidian-rag-"
demo_url: null
status: "已完成"
---
```

### 3.4 时间线 JSON

```json
{
  "id": "10",
  "date": "2026-07-01",
  "title": "完成智学 Obsidian AI 插件",
  "description": "描述文字…",
  "type": "project",
  "link": "/projects/zhi-xue",
  "created_at": "2026-07-01"
}
```

事件类型：`milestone`（里程碑）、`project`（项目）、`article`（文章）、`award`（获奖）、`internship`（实习）、`learning`（学习）。

## 四、部署

### 4.1 自动部署（推荐）

仓库连接了 Vercel，推送到 `main` 分支自动构建部署。

```bash
# 只改了内容文件（推荐用这个脚本）
./publish.sh "新增了关于机器学习的文章"

# 改了代码（如组件、样式）
git add -A
git commit -m "描述你的改动"
git push
```

### 4.2 部署后

Vercel 构建约 1-2 分钟，完成后自动上线到 `starrylovetbao.cloud`。

## 五、主题系统

右下角的 ⚙ 按钮提供主题切换：

| 主题 | 背景色 | 说明 |
|------|--------|------|
| 暖黄 | `#fefce8` | **默认主题**，温暖护眼 |
| 浅色 | `#fafafa` | 干净极简 |
| 灰色 | `#f4f4f5` | 低饱和中性 |
| 深色 | `#18181b` | 完整暗色模式（文字/卡片/边框全适配） |

- 主题选择**自动记忆**到浏览器
- 刷新页面**无闪烁**（blocking script 在 hydration 前注入）
- ⚙ 按钮可拖动到任意位置，刷新后**位置记忆**

## 六、首页联系方式

首页 Hero 区底部（`Connect` 标题）展示个人联系方式和社交媒体链接，位于 `src/components/HeroSection.tsx`。

### 已有平台

| 平台 | 显示内容 |
|------|---------|
| GitHub | `github.com/wanghongyu666qiang` |
| Email | `why17573315302@gmail.com` |

### 添加新平台

在 HeroSection.tsx 的 Connect 区域 `<div className="flex flex-col gap-2.5">` 内，仿照以下模板新增：

```tsx
{/* 平台名 */}
<a
  href="链接地址"
  target="_blank"
  rel="noopener noreferrer"
  className="group inline-flex items-center gap-2 text-sm text-text-primary hover:opacity-80 transition-opacity duration-200 w-fit"
>
  <svg className="w-4 h-4 shrink-0" viewBox="0 0 24 24" fill="currentColor">
    {/* SVG 图标 path */}
  </svg>
  显示文字
</a>
```

**三个要素**：SVG 图标 + 链接 URL + 显示文字。图标可去 [Simple Icons](https://simpleicons.org) 或 [Lucide](https://lucide.dev) 复制 SVG path。

## 七、快速参考

| 任务 | 操作 |
|------|------|
| 写文章 | 后台 `/admin/posts/new` 或直接在 `src/content/posts/` 创建 `.md` |

| 更新时间线 | 后台 `/admin/timeline` 或编辑 `timeline.json` |
| 改密码 | 修改 `.env.local` 的 `ADMIN_PASSWORD` |
| 改主题 | 右下角 ⚙ → 背景颜色 |
| 看统计 | 后台 `/admin/dashboard` |
| 改联系方式 | 编辑 `src/components/HeroSection.tsx` |
| 部署上线 | `./publish.sh "提交信息"` |

## 八、维护建议

- 文章内容写在 Markdown 里，格式齐全（KaTeX 数学、Mermaid 图表、代码高亮）
- 时间线定期更新，不只是成果也加学习过程
- 新项目及时挂到首页精选区
- 发布前先在 `npm run dev` 预览
